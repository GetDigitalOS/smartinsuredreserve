# SmartInsuredReserve — Autonomous Build Backlog

SmartInsuredReserve is a Next.js 14 / React 18 / TypeScript single-page calculator that helps users optimize insurance deductibles by building a self-insurance reserve fund. The app is fully functional as a working component (`InsuranceOptimizer.tsx`) but has zero test coverage on its core projection logic, relies on `any` types throughout, and contains scaffold-remnant helpers unrelated to the domain. This spec drives extraction of the projection engine into a testable library, full unit test coverage, input validation, and a scenario-persistence layer.

Done-gate: `npm test`

## Build

- [ ] **Extract projection engine into `src/lib/projection.ts`**: Pull the `useMemo` calculation loop out of `InsuranceOptimizer.tsx` into a pure function `calculateProjection(inputs: ProjectionInputs): ProjectionRow[]`. Define `ProjectionInputs` and `ProjectionRow` TypeScript interfaces (replacing all `any` types) in `src/lib/types.ts`. Update `InsuranceOptimizer.tsx` to import and call `calculateProjection` in place of the inline `useMemo` body. Add `tests/projection.test.ts` with at least 6 unit tests covering: year-1 premium computation, deductible upgrade threshold logic, reserve compounding, cumulative savings accumulation, zero-reserve start, and 1-year projection edge case.
